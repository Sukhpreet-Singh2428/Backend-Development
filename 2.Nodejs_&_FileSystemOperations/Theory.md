## theory :

- [ ] js se backend nahi ban sakta, kyuki js ke paas wo functionalities nahi hai jisne backend banta hai
- [ ] ryan dahl, isne socha js se backend banna chahiye
- [ ] google chrome ka v8 engine ka code open source hai and ryan dahl us code mein chhedkhanni kar raha hai
- [ ] kyuki chrome ka v8 engine bana hai c++ mein
- [ ] fir usne socha ki humein to js mein code karna hai
- [ ] ryan dahl ne v8 engine ke code ko js ke code se wrapped kar diya
- [ ] hum js ka code likhege jo ki wrapper layer of js receive karegi and wo code v8 engine ke c++ modules ke saath ek server create krega
- [ ] so this whole js wrapper + v8 engine in c++, this whole think is called as Node.js


==node.js is a js runtime environment==

npm is like app store or play store and in more detail we can say that package store

so basically we can upload our code bases in npm, those are called packages and fir logg usse download karke use kar sakte hai unke code bases mein


==npm (Node Package Manager) is the world's largest software registry and a crucial tool for JavaScript developers, serving as a command-line utility to install, share, and manage reusable code packages (libraries, frameworks, tools) for Node.js and frontend projects, making development efficient by providing access to millions of pre-written modules. It's the default package manager for Node.js and central to the JavaScript ecosystem, helping manage project dependencies through== `package.json` ==files.==
